Tech Stack & Architecture
  • Frontend: HTML + Bootstrap 5, custom CSS, Chart.js
  • State: Browser LocalStorage (per-browser, per-user)
  • Backend: Flask (Python) serving pages + /api/predict
  • ML: Optional scikit-learn model via fraud_model.pkl
  • Persistence (Dev): LocalStorage + optional server snapshot JSON

Flow:

  • UI form ‚Üí JSON payload ‚Üí /api/predict
  • Model/Dummy scorer ‚Üí probability + label
  • UI renders results + charts + saves session
  • (Optional) Auto-backup ‚Üí data/users.json & data/predictions.json
Data & Privacy (Dev Mode)
  • By default, data aapke browser ke LocalStorage me store hota hai.
  • Browser clear/Incognito/use of another browser ‚Üí data different/erase ho sakta hai.
  • Auto-backup enabled hone par JSON files yahan milengi: data/users.json, data/predictions.json.
  • Production use ke liye proper database (Mongo/Postgres) + server-side auth recommended.
  • PII collect karne se pehle compliance aur encryption policies follow karein.
Limitations & When To Scale
  • LocalStorage per-browser hota hai; multi-user, multi-device sync ke liye DB zaroori hai.
  • High volume logging, audit trails, RBAC, alerts ‚Üí use a real backend database.
  • Model lifecycle (retraining, versioning) ke liye file-based pickle se aage MLOps stack consider karein.
Roadmap
  • Role-based access (analyst, auditor) with granular permissions
  • Batch upload + scoring for CSV transactions
  • Explainability (feature impact) for each prediction
  • Alerts & webhooks for high-risk flags
  • Pluggable DB adapters (MongoDB / Postgres)
FAQs

Q1. Data kahan save hota hai?
Default: LocalStorage. Agar auto-backup enabled hai to /data/*.json me server-side snapshots bhi bante hain.

Q2. Apna ML model kaise add karun?
fraud_model.pkl ko project root me rakhein. Flask automatically load kar lega, warna dummy scorer use hoga.

Q3. Production me kaise le jaayen?
Real DB + server auth + secure storage + logging + monitoring add karke deploy karein (Render/Heroku/VM/K8s).
